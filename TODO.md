## 未実装機能 (優先順)

### P0: 代入・パス操作 (68件) — 使用頻度: 極めて高い

jq でのデータ変換の中核機能。query だけでなく mutation に必須。`.foo = "bar"` や `.[] |= . + 1` は jq ユーザーが日常的に使うパターン。

**必要な実装:**
- `eval_path(filter, input)` — フィルタからパス (例: `["foo", 0, "bar"]`) を抽出
- `setpath(input, path, value)` — 指定パスに値を設定
- `delpaths(input, paths)` — 指定パスの値を削除
- パーサーで `=`, `|=`, `+=`, `-=`, `*=`, `/=`, `%=`, `//=` を認識

**テスト内訳:**
- `=` 代入 (26件): `.foo = 1`, `.[0] = "x"`, `.foo.bar = .baz`
- `|=` 更新代入 (14件): `.foo |= . + 1`, `.[] |= select(. > 2)`
- `setpath` (6件): `setpath(["a","b"]; 1)`
- `del` (5件): `del(.foo)`, `del(.[2:4])`
- `path` 式 (8件): `path(.foo[0,1])`, `path(.[] | select(.>3))`
- `pick` (4件): `pick(.a.b.c)`
- `delpaths` (3件): `delpaths([["a"]])`
- `//=` 代替代入 (1件): `.[] //= .[0]`
- `getpath` 高度な使い方 (1件)

**実装見積もり:** ~380行

**使用例 (典型的な jq ワークフロー):**
```bash
# 設定ファイルの値を変更
cat config.json | jq '.database.host = "localhost"'

# 配列の各要素を更新
echo '[1,2,3]' | jq '.[] |= . * 2'

# 不要なフィールドを削除
echo '{"name":"a","tmp":1}' | jq 'del(.tmp)'

# 条件付き更新
echo '[1,2,3,4,5]' | jq '(.[] | select(. > 3)) |= . * 10'
```

### P1: 文字列補間 (10件) — 使用頻度: 高い

- パーサーで `StringInterp(Array[StringPart])` ノードを構築
- eval で各パートを評価して結合

**実装見積もり:** ~100行

## 既知の互換性差異 (修正困難)

プラットフォーム (IEEE 754 double / JS 文字列) に起因し、修正不可能:

- 巨大整数 (>2^53) が `Infinity` になる
- `-0` が `0` として出力される
- NUL 文字 (`\u0000`) を含む文字列の `contains` が不正確
- `fromjson` のエラーメッセージが jq と異なる
- エラーメッセージの大文字/小文字が jq と異なる (`Cannot` vs `cannot`)
